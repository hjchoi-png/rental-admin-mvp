# 코드 품질 점검 워크플로우

지정된 파일 또는 영역의 코드를 종합 점검하라.
단순 스타일이 아니라, 실제 문제를 찾아내는 것이 목표.

## 입력
$ARGUMENTS

(입력이 없으면 최근 변경된 파일들을 자동 감지하여 점검)

## 점검 항목

### 1. 보안 (secrets-management, auth-implementation-patterns 스킬)
- API 키, 시크릿이 코드에 하드코딩되어 있지 않은지
- RLS 정책이 적절한지
- 인증/인가 검증이 빠져있는 곳은 없는지
- SQL 인젝션, XSS 가능성

### 2. 성능 (vercel-react-best-practices, sql-optimization-patterns 스킬)
- N+1 쿼리
- 불필요한 re-render
- 누락된 인덱스
- 번들 크기에 영향을 주는 import

### 3. 타입 안전성 (typescript-advanced-types 스킬)
- any 사용 여부
- 타입 단언(as) 남용
- 런타임에 터질 수 있는 타입 불일치

### 4. 에러 처리 (error-handling-patterns 스킬)
- catch 블록에서 에러를 삼키고 있지 않은지
- 사용자에게 적절한 에러 메시지를 보여주는지
- 외부 API 실패 시 그레이스풀 디그레데이션

### 5. Next.js 패턴 (next-best-practices, nextjs-app-router-patterns 스킬)
- Server/Client Component 경계가 적절한지
- 데이터 페칭 패턴이 올바른지
- 캐싱/재검증 설정

## 출력 형식

각 발견 사항을 아래 형식으로 보고:
- **심각도**: critical / major / minor
- **위치**: 파일:라인
- **문제**: 무엇이 잘못되었는지
- **해결**: 어떻게 고쳐야 하는지
- **자동 수정 제안**: 사용자 동의 시 바로 수정 가능
