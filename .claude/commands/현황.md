# 프로젝트 현황 리포트

프로젝트의 현재 상태를 종합적으로 파악하여 보고하라.

## 수집할 정보

### 1. 코드 현황
- 전체 파일 수, 주요 디렉토리별 파일 수
- TypeScript 에러 여부 (npx tsc --noEmit)
- 빌드 성공 여부

### 2. 진행 상태
- docs/progress.md 최신 세션 내용
- CLAUDE.md 로드맵 대비 완료/진행 중/미착수 현황

### 3. 코드 품질 간이 점검
- `any` 타입 사용 횟수
- `TODO`/`FIXME` 코멘트 목록
- 미구현 영역 (주석으로 표시된 부분)

### 4. 의존성 현황
- package.json 주요 의존성 버전
- 보안 취약점 (npm audit 결과)

## 출력 형식

한국어로, 비개발자가 이해할 수 있는 수준으로 작성.
기술 용어에는 괄호 안에 쉬운 설명 추가.

```
## 프로젝트 현황 리포트 (날짜)

### 완료된 것
- ...

### 진행 중
- ...

### 다음 할 일
- ...

### 주의 사항
- ...

### 전체 건강도: [좋음/보통/주의]
```
