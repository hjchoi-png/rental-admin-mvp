# 버그 수정 워크플로우

보고된 버그를 체계적으로 분석하고 수정하라.
추측으로 코드를 수정하지 말고, 근본 원인을 먼저 파악하라.

## 입력
$ARGUMENTS

## 단계

### 1단계: 재현 및 근본 원인 분석 (systematic-debugging 스킬)
- 버그 증상을 정확히 파악
- 관련 코드 탐색 (Grep, Read 활용)
- 에러 로그/스택 트레이스 분석
- 가설 수립: 원인 후보 2-3개 나열
- 각 가설 검증 (코드 추적, 데이터 확인)
- **근본 원인 확정 후 사용자에게 보고** (여기서 반드시 멈추고 설명)

### 2단계: 수정 (test-driven-development 스킬)
- 가능하면 실패하는 테스트를 먼저 작성 (버그 재현)
- 최소한의 변경으로 근본 원인 수정
- 테스트 통과 확인
- 관련 기술 스킬 자동 적용:
  - DB 관련 → supabase-postgres-best-practices
  - Next.js → next-best-practices
  - 타입 → typescript-advanced-types
  - 에러 처리 → error-handling-patterns

### 3단계: 검증 (verification-before-completion 스킬)
- 빌드 성공 확인
- 관련 테스트 전체 실행
- 부작용(사이드 이펙트) 없는지 확인
- 수정 내용 요약 제공
- docs/progress.md에 기록
