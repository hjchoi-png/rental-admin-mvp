# 새 기능 구현 워크플로우

사용자가 요청한 새 기능을 아래 단계를 **순서대로** 실행하여 구현하라.
각 단계에서 해당 스킬의 베스트 프랙티스를 자동으로 참조하라.

## 입력
$ARGUMENTS

## 단계

### 1단계: 브레인스토밍 (brainstorming 스킬)
- 요청을 분석하고 프로젝트 컨텍스트(CLAUDE.md, 관련 코드)를 탐색
- 가능한 접근법 2-3개를 제시
- 각 접근법의 장단점을 설명
- **사용자에게 방향 확인 받기** (여기서 반드시 멈추고 물어볼 것)

### 2단계: 설계 (writing-plans 스킬)
- 선택된 접근법을 구체적 구현 플랜으로 분해
- 수정할 파일 목록, 새로 만들 파일 목록 명시
- DB 변경이 필요하면 마이그레이션 포함
- **사용자에게 플랜 확인 받기**

### 3단계: 구현 (executing-plans 스킬)
- 플랜을 배치(batch) 단위로 구현
- 각 배치 완료 시 중간 진행 보고
- 관련 기술 스킬 자동 적용:
  - DB 작업 → supabase-postgres-best-practices
  - Next.js 코드 → next-best-practices, nextjs-app-router-patterns
  - 컴포넌트 → vercel-composition-patterns
  - 타입 → typescript-advanced-types
  - UI → frontend-design, tailwind-design-system
  - API → api-design-principles
  - 에러 처리 → error-handling-patterns

### 4단계: 검증 (verification-before-completion 스킬)
- 빌드 성공 확인 (npx next build 또는 npx tsc)
- 관련 테스트 실행 (있는 경우)
- 변경사항 요약 제공
- docs/progress.md에 작업 기록
