# STR 공통 용어 정의

> 모든 정책 문서, CS 스크립트, 개발 명세, AI 프롬프트에서 **단일 기준**으로 사용되는 용어 사전입니다.
> 새로운 용어를 사용할 경우 반드시 이 문서에 추가한 후 사용합니다.

---

## 서비스 정체성 관련 용어

| 용어 | 정의 | 주의사항 |
|------|------|---------|
| 단기임대 (STR) | 일정 기간(주 단위 이상) 거주를 목적으로 하는 임대차 계약 구조의 서비스. 숙박업과 구분된다. | ≠ 숙박, ≠ 호텔, ≠ 에어비앤비 숙박 |
| 일시사용 임대차 | 단기 거주를 목적으로 한 임대차 계약 형태. 숙박 서비스가 아닌 거주 계약이다. | 법적 근거: 민법 임대차 |

### 사용 금지 용어 → 올바른 용어

| 사용 금지 | 올바른 용어 | 이유 |
|----------|-----------|------|
| 숙박 | 거주 / 임대 | 숙박업이 아님 |
| 체크인 | 입주 | 호텔 용어 사용 금지 |
| 체크아웃 | 퇴실 / 퇴거 | 호텔 용어 사용 금지 |
| 연장 | 신규 계약 | 연장 개념 없음, 항상 신규 계약 체결 |
| 호스팅 | 매물 운영 | 숙박 플랫폼 용어 배제 |
| 1박 / 연박 | 1일 / 거주 기간 | 숙박업 오인 표현 금지 |
| 어메니티 | 비품 / 제공 물품 | 숙박업 오인 표현 금지 |
| 객실 / 프론트 | 공간 / (해당 없음) | 숙박업 오인 표현 금지 |

## 계약 구조 관련 용어

### 계약 상태 흐름

```
예약 요청 → 예약 승인 → 결제 대기 → 계약 체결 → 거주 중 → 퇴거 요청 → 퇴거 확인 → 계약 종료
```

| 용어 | 정의 |
|------|------|
| 예약 요청 | 게스트가 특정 기간에 대해 계약 의사를 표시하는 단계 |
| 예약 승인 | 호스트가 해당 기간에 대해 계약 진행 의사를 확인하는 단계 |
| 결제 대기 | 승인 이후 게스트가 결제를 완료하기 전 상태 |
| 계약 체결 | 결제가 완료되어 계약이 성립된 상태 |
| 거주 중 | 입주일이 시작되어 실제 공간 사용이 이루어지고 있는 상태 |
| 퇴거 요청 | 게스트가 계약 종료 의사를 시스템에 표시하는 단계 |
| 퇴거 확인 | 호스트가 공간 점검을 완료했음을 표시하는 단계 |
| 계약 종료 | 보증금 반환까지 완료되어 계약 관계가 정리된 상태 |

## 금액 구조 관련 용어

| 용어 | 정의 | 비고 |
|------|------|------|
| 임대료 | 공간 사용에 대한 기본 비용 | 주/일 단위 산정. 할인 적용 대상. |
| 관리비 | 전기·수도·가스·인터넷·건물 관리비 포함 비용 | 주/일 단위 산정 |
| 청소비 | 입실 전(퇴실 후) 1회 수행되는 청소·세탁 비용 | 건 단위 1회, 연장 시 재부과 없음 |
| 보증금 | 파손·분실·관리비 초과 등에 대비한 예치금 | **30만원 고정**, 플랫폼이 보관 |
| 위약금 | 계약 취소 시점에 따라 발생하는 임대료 기준 차감 금액 | 위약금 정책표 참조 |
| 플랫폼 수수료 | 거래 인프라 이용에 대한 수수료 | 호스트 3%, 게스트 9% (VAT 제외) |
| 장기 할인 | 계약 기간에 따른 임대료 할인율(%) | 호스트가 설정, 임대료에만 적용 |
| 즉시입주 할인 | 입주 임박 매물에 대한 정액 할인(만원 단위) | 호스트가 설정, 임대료에만 적용 |

## 행위/상황 관련 용어

| 용어 | 정의 | 주의사항 |
|------|------|---------|
| 조기 퇴거 | 거주 시작 이후 계약 기간을 채우지 않고 종료하는 것 | = 해지 개념 (취소 아님) |
| 신규 계약 | 기존 계약 종료 후 동일 매물을 추가로 이용하기 위한 새로운 계약 | ⚠️ 자동 연장 없음. 항상 신규 계약 체결 필요 |
| 보증금 보류 | 퇴거 점검 과정에서 차감 여부 판단을 위해 반환을 일시적으로 멈추는 행위 | 최대 10일 |
| 차감 사유 | 시설 파손, 물품 분실, 관리비 초과 등 보증금 일부를 정산해야 하는 사유 | |
| 중도 퇴거 | 거주 중 계약 기간을 채우지 않고 퇴거하는 행위 | 미사용 임대료 환불 없음 |
| 자동 퇴실 처리 | 퇴실 예정일 기준 24시간 내 미요청 시 시스템이 자동으로 퇴실 처리하는 것 | |
| 자동 퇴거 확인 | 퇴거 요청 후 24시간 내 호스트 미응답 시 자동으로 퇴거 확인 처리되는 것 | |

## 역할 정의

| 용어 | 정의 |
|------|------|
| 게스트 | 공간을 일정 기간 사용하는 계약 당사자 |
| 호스트 | 공간을 제공하는 계약 당사자 |
| 플랫폼 | 계약·결제·보증금 관리 등 **거래 인프라를 제공**하는 중개 주체. 계약 당사자가 아니며, 분쟁에 개입·판단하지 않음. |

## 취소/해지 구분

| 용어 | 정의 | 적용 시점 |
|------|------|----------|
| 결제 전 취소 | 예약 승인 후, 결제 완료 전에 예약을 철회하는 행위 | 결제 대기 상태 |
| 계약 취소 | 계약 체결 후, 입주 전에 계약을 해소하는 행위. 위약금 발생 가능 | 계약 체결 ~ 입주일 이전 |
| 해지 (중도 퇴거) | 거주 중 계약을 종료하는 행위 | 거주 중 |

> **핵심**: "취소"와 "해지"는 다른 개념. 거주 시작 전 = 취소, 거주 시작 후 = 해지(중도 퇴거)

## 정책 분류 용어

| 용어 | 정의 | 예시 |
|------|------|------|
| 기준 정책 | 무엇이 허용되고 어떻게 계산되는가 | 환불 기준, 플랫폼 수수료 |
| 운영 정책 | 플랫폼이 어떤 방식으로 운영되는가 | 계약 성립 시점, 예약 승인 구조 |
| 절차 정책 | 문제가 생기면 어떤 순서로 처리되는가 | 분쟁 처리 흐름, 증빙 제출 방식 |
