2. 계약(예약) 기간

**결론 (SRT 팀 합의 의견) : 주로 할경우 1주 ~12주 // 월로 할 경우, 1개월~3개월까지**
- 최종 팀 내부 의견으로는 원오빌 사업에 충돌 우려도 있기때문에 6개월은 지향하고 최소 1주부터 최대 3개월까지 진행 함을 의견 드림.

1) 법적 기준
- 통상적으로 단기임대는 최소 1주 ~ 최대 6개월 범위로 해석 가능
- 다만 실무적·규제 리스크 관점에서의 안전 구간은 3개월(90일)까지로 보는 것이 합리적일 수 있음

2) Reference - 타사 플랫폼 운영 기준
- 삼삼엠투: 최대 3개월(90일) *13주 이상은 기간 연장 신청으로 진행 안내
- 리브애니웨어: 최대 약 5.9개월(179일)

4) 참고 - 추가 아이디어 제안 (실무 경험 기반)
① 월 단위 결제 옵션 제공
- 경쟁 서비스는 대부분 주 단위 결제가 메인, 월 단위 결제 옵션이 있으면  유저 입장에서 가격·기간 이해가 쉬움
- 1개월 이상 체류 시 심리적 부담 감소하고, 중기 체류 니즈를 자연스럽게 흡수 가능
② 1주 이상 체류 시 ‘일 단위 연장’ 허용
- 실무 경험상“ 1주 + 1~2일 더 머물고 싶다”는 니즈가 빈번
- 기존 플랫폼(삼삼엠투)는 주 단위 고정 구조로 인해 불편 발생
-  제안: 최소 1주(7일) 이후는 일 단위로 유연하게 설정 가능
③ 날짜 직접 선택 방식 제안
- ‘주 단위 패키지 선택’ 대신 날짜 직접 선택 (최소 7일 이상) 구조, 최대 기간은 3개월 미만으로 제한
- 장점 : 실제 거주 일정과 UX가 자연스럽게 맞음, 연장/조정 니즈 흡수 가능, 규제 리스크 관리 용이


3. 일단위 연장 계약 가격 정책

**결론 (SRT 팀 합의 의견) : 일 단위 연장 금액 산식: (임대료+관리비)÷7**
- 최종 팀 내부 의견으로는 계산 대상: (임대료+관리비)  *청소비는 1회성 비용으로 계산에서 제외해야 함
- (임대료 + 관리비) * 수수료%

**기간정책**
- 최소 계약 기간: 7일 (1주)
ㄴ 신규 계약 시, 최소 7일 충족하는 경우 1일 단위 같이 예약 가능
ㄴ 연장 계약 시는 1일 단위가 아니라 다시 7일 계약 + 1일 단위 같이 예약 가능
- 계약(예약) 기준 위에 내용 정리
ㄴ 최초 예약 : 최소 1주(7일)
ㄴ 연장 예약 : 최소 1주(7일)
ㄴ 공통 예시 1주(필수) = 7일 + 2일 = 9일
