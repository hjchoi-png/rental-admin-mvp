# 정책-3. 운영 기준 (채팅·금지 키워드)

> 채팅 닉네임 정책, 금지 키워드 검증 규칙, 자동 안내 메시지에 관한 운영 기준입니다.

---

## 채팅 닉네임 / 채팅방 이름 정책

### 게스트 닉네임 정책

| 항목 | 내용 |
|------|------|
| 생성 시점 | 채팅방 생성 시 |
| 생성 규칙 | 직방 계정의 닉네임을 그대로 사용 |
| 예외 처리 | 직방 닉네임이 없는 경우(전체 회원의 약 0.6%) → 채팅방 생성 시 랜덤 닉네임 자동 부여 |
| 닉네임 목적 | Post-MVP 리뷰 등에서 게스트 식별을 일관되게 유지하기 위함 |

### 호스트 닉네임 정책

| 항목 | 내용 |
|------|------|
| 생성 시점 | 호스트 전환 시 (온보딩 과정) |
| 생성 규칙 | 사용자가 직접 입력한 닉네임을 사용 |
| 동기화 | 닉네임 변경 시 기존에 생성된 **모든 채팅방의 호스트 이름이 최신 값으로 일괄 반영** |
| 닉네임 목적 | 채팅방 식별 및 우수 호스트 소개 등 활용 |

### 채팅방 이름 정책

| 노출 대상 | 채팅방 이름 구성 |
|----------|--------------|
| 게스트에게 노출 | 호스트 닉네임 + 매물 이름 |
| 호스트에게 노출 | 게스트 닉네임 + 매물 이름 |

**Sendbird 채팅방 기본 이름**: 매물 이름을 기준으로 생성

### 채팅방 이름 동기화 제약 사항

| 변경 항목 | 동기화 여부 |
|----------|-----------|
| 호스트/게스트 닉네임 변경 | ✅ 채팅방 이름의 닉네임 영역이 최신 값으로 반영됨 |
| 매물 이름 변경 | ❌ 이미 생성된 채팅방의 매물 이름은 변경되지 않음 (생성 시점 기준 고정) |

> 사유: 특정 매물에 연결된 모든 채팅방을 조회하고 갱신하는 구조가 기술적으로 불가

### 금지 키워드 정의 및 목적

**정의**: 플랫폼 내 매물 상세 설명에 유저가 직접 입력하는 텍스트 영역에서 발생할 수 있는 부적절한 키워드를 감지해서, 위험성을 알려주고 플랫폼 거래를 보호

**목적**:
- 서비스 신뢰도 및 브랜드 톤앤매너 유지
- 플랫폼 이탈 방지 및 수익 구조 보호
- 법적·운영적 리스크 최소화
- 분쟁 데이터의 객관화

## 자동 안내 메시지 (검수 결과)

### 매물 검수 결과 안내 (반려/보완 시)

```
[단기임대 매물 검수 결과 안내]

등록 기준에 부합하지 않는 내용이 확인되어
해당 매물의 노출이 일시적으로 제한되었습니다.

자세한 반려 사유와 수정이 필요한 항목은
아래 [반려 사유 확인] 메뉴에서 확인해 주세요.

안내된 내용을 수정하시면
재검수 후 다시 노출이 가능합니다.

☑️ 유의해 주세요
등록 기준에 맞지 않는 내용은 사전 안내 없이 노출이 제한될 수 있습니다.
자세한 기준은 매물 등록 가이드를 참고해 주세요.
```

### 안내 멘트 예시 (사용자 표시용)

- 직거래/개인정보 공유 경고:
  "직거래나 개인 정보 공유는 사기 피해의 위험이 큽니다. 앱을 통해 안전한 결제와 채팅 시스템을 이용해야만 나중에 문제가 생겼을 때 보호받으실 수 있어요."

- 숙박업 오인 표현 경고:
  "조식, 어메니티 등의 단어는 숙박업으로 오인될 수 있습니다. 본 서비스는 일시사용 임대차 계약이므로 임대차 용어(입주, 퇴거, 비품 등)로 수정을 권장드려요."
