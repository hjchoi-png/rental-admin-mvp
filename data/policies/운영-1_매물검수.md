# 운영-1. 매물 검수 정책

> 매물 등록부터 검수, 신고 처리까지 전체 프로세스와 기준을 정의합니다.

---

## 매물 검수 대전제

### 1) 시스템이 메인, 사람은 예외 처리만

- 매물 검수는 시스템과 자동화 로직이 1차로 판단
- 문제 발생 가능성이 높은 매물만 사람이 개입하는 구조
- 자동 검수 통과 매물은 기본적으로 자동 승인
- **자동 승인 비율 목표: 90% 이상**
- 관리자는 전체를 보는 역할이 아니라 **예외를 처리하는 역할**

### 2) 사전 완벽 검수보다 사후 빠른 대응 우선

- 모든 문제를 사전에 막는 것이 목표가 아님
- 신고·이탈·이상 징후 등 사후 신호를 빠르게 감지하고 즉시 조치하는 것이 목표
- 매물 신고 기능은 검수의 핵심 신호로 활용
- 신고 발생 시: 즉시 숨김 또는 재검수 대상 전환, 우선 처리 큐로 이동

### 3) 신고 대상 매물은 중요도 기준으로 처리

- 신고는 등록 순서가 아니라 **리스크 신호를 기준**으로 우선 처리
- 우선 검수 대상: 매물 신고 발생 매물, 중복 의심 매물, 고가/대형/특이 매물
- 일반 매물은 자동 검수 통과 시 별도 개입 없이 승인

### 4) 검수 기준은 일관되고 재사용 가능해야 함

- 검수 기준은 운영자 개인의 판단이 아니라 어드민에 정의된 **표준 항목과 사유**를 기준으로 처리
- 반려 사유는 어드민에서 관리
- 알림톡·매물 리스트·CS 응대에서 동일한 기준 사용

### 5) 검수의 목표는 '승인 정확도'가 아니라 '운영 효율'

- 운영 리소스를 최소화하면서 서비스 품질을 유지하는 것이 목적
- 관리자 검수 대상 비율을 지속적으로 감소시키는 구조 지향

### 6) 모든 검수 결과는 다음 자동화를 위한 데이터로 활용

- 반려 사유, 신고 유형, 재검수 이력 등을 데이터로 축적

## 검수 목적 (6가지)

1. **진실성 확보**: 허위·과장·오해 유발 정보를 차단
2. **안전/리스크 최소화**: 불법·규정 회피, 권한 불명확, 사기 가능성을 선제적으로 차단
3. **플랫폼 내 거래 유도**: 외부 연락/외부 결제 유도를 금지하고 플랫폼 내 절차로 거래가 이루어지게 함
4. **명확성(분쟁 예방)**: 주소·조건·요금·룰이 명확히 기재되도록 하여 분쟁을 예방
5. **품질(신뢰 확보)**: 사진·설명 등 콘텐츠의 최소 품질 기준을 충족시켜 사용자 신뢰를 확보
6. **사후 통제 가능성**: 승인 이후에도 문제 발생 시 즉시 노출정지 및 재검증이 가능하도록 운영

## 매물 검수 프로세스

### 일반 매물 등록

```
매물 등록
  → 매물 검수 (시스템 규칙 → AI Agent)
  → 검수 완료 (승인) → 노출 중
  → 검수 보완 요청 → 시스템 알림 → 호스트가 수정 → 재검수
    → 통과 → 검수 완료 → 매물 노출
    → 2차 반려 → 사람 개입
  → 반려 → 노출 불가 → 호스트 수정 → 재검수
    → 통과 → 매물 노출
    → 2차 반려 → 사람 개입
```

### 신고 접수 시

```
매물 노출 중 → 신고 접수
  → 시스템 규칙 + AI Agent 검토
    → 문제 없음 → 처리 완료 → 노출 중
    → 보완 필요 → 알림 → 호스트 수정 → 재검수
    → 위반 확정
      → 노출 중지 → 호스트 수정 → 재검수
      → 사람 개입 필요 시 → CS 응대
```

## 사후 매물 신고 처리

사후 신고: 검수 완료 후 노출 중인 매물에 대해 이용자가 문제를 제기하는 것.

### 신고 처리 기준

| 검수 영역 | 검수 항목 | 상태값 | 신고 처리 기준 | 전제 조건 |
|---------|---------|------|------------|---------|
| 주소 | 다른 동호수 | 검수 반려 | 계약 시 진행된 주소가 완전히 다른 경우 | 게스트가 안내 문자 제출 |
| 주소 | 다른 주소 | 검수 반려 | 계약 시 진행된 호수와 호스트 안내 호수가 다른 경우 | 게스트가 안내 문자 제출 |
| 사진 | 공간 허위 | 검수 반려 | 올린 매물 사진과 완전 다른 방인 경우 | 게스트가 찍은 사진 필요 |
| 사진 | 공간 불일치 | 검수 반려 | 방 개수·화장실 숫자 등이 다른 경우 | 게스트가 찍은 사진 필요 |
| 사진 | 옵션 허위 | 검수 반려 | 침대·에어컨·세탁기 등 대형 가구/가전 옵션이 없는 경우 (소형 제외) | 게스트가 찍은 사진 필요 |
| 사진 | 공간 정보 부족 | 검수 보완 | 일부 공간 사진 누락 | 게스트가 추가 정보 요청한 경우 |
| 사진 | 옵션 정보 부족 | 검수 보완 | 공간과 상관없는 사진이 많은 경우 | 게스트가 추가 정보 요청한 경우 |
| 계약 | 직거래 유도 | 검수 반려 | 호스트가 직거래 유도를 한 경우 | 게스트가 채팅 내용 제출 |
| 계약 | 외부 연락 유도 | 검수 반려 | 호스트가 앱 내 채팅 외 외부 채팅(카톡 등) 요구 | 게스트가 채팅 내용 제출 |

### 관리 대상 호스트 분류 기준

> 동일 매물 기준으로 신고가 **3회 이상 접수되어 위반으로 확정**된 경우, 해당 매물을 등록한 호스트는 **관리 대상 호스트**로 분류합니다.
