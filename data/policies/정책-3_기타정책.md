# 정책-3

[정책-3. 그 외 여러가지]


1. 단기임대 채팅 닉네임/채팅방 이름 정책

1) 게스트 닉네임 정책
생성 시점: 채팅방 생성 시
생성 규칙: 직방 계정의 닉네임을 그대로 사용

예외 처리
직방 닉네임이 없는 경우(전체 회원의 약 0.6%)
채팅방 생성 시 랜덤 닉네임을 자동 부여
닉네임 생성 목적: Post-MVP 리뷰 등에서 게스트 식별을 일관되게 유지하기 위함

2) 호스트 닉네임 정책
생성 시점: 호스트 전환 시 (온보딩 과정)
생성 규칙: 사용자가 직접 입력한 닉네임을 사용
동기화 정책 (중요)
호스트 닉네임은 단일 값으로 관리
닉네임 변경 시
기존에 생성된 모든 채팅방의 채팅방 리스트 / 채팅방 상세 내 호스트 이름 → 최근 저장된 닉네임으로 일괄 반영 됨
닉네임 생성 목적: 채팅방 식별 및 우수 호스트 소개 등 활용

3) 채팅방 이름 정책
Sendbird 채팅방의 기본 이름: 매물 이름을 기준으로 생성
게스트에게 노출되는 채팅방 이름 : 호스트 닉네임 + 매물 이름
호스트에게 노출되는 채팅방 이름 : 게스트 닉네임 + 매물 이름

4) 채팅방 이름 동기화 정책 (제약 사항)
닉네임 영역
호스트 / 게스트 닉네임 변경 시 채팅방 이름의 닉네임 영역은 최신 값으로 반영
매물 이름 변경 시 이미 생성된 채팅방의 매물 이름은 변경되지 않음
사유: 특정 매물에 연결된 모든 채팅방을 조회하고 갱신하는 구조가 불가
결론: 채팅방 이름 내 매물명은 생성 시점 기준 값으로 고정. 채팅방 이름 내 닉네임은 최신 값으로 반영
