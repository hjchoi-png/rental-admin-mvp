# 개발 서버 실행 문제 해결 가이드

## 문제 상황
`EPERM: operation not permitted` 오류가 발생하는 경우, macOS의 네트워크 권한 문제입니다.

## 해결 방법

### 방법 1: 시스템 설정에서 권한 부여 (권장)

1. **시스템 설정** 열기
2. **개인 정보 보호 및 보안** > **방화벽** 이동
3. 방화벽이 켜져 있으면:
   - **방화벽 옵션** 클릭
   - Node.js 또는 터미널 앱이 차단 목록에 있으면 허용으로 변경

4. **개인 정보 보호 및 보안** > **네트워크** 이동
5. 터미널 앱(또는 사용 중인 터미널)에 체크가 되어 있는지 확인
6. 없으면 체크박스를 활성화

### 방법 2: 다른 터미널 앱 사용

- iTerm2, Alacritty 등 다른 터미널 앱을 사용해보세요
- 새로운 터미널 앱은 처음 실행 시 네트워크 권한을 요청합니다

### 방법 3: 방화벽 일시 비활성화

1. **시스템 설정** > **개인 정보 보호 및 보안** > **방화벽**
2. 방화벽을 일시적으로 끄고 서버 실행
3. 작동하면 방화벽 설정을 조정

## 현재 설정

- 개발 서버 포트: **50000**
- 접속 주소: **http://127.0.0.1:50000**
- 커스텀 서버 파일: `server.js`

## 실행 명령어

```bash
npm run dev
```

또는

```bash
npm run dev:next
```
