## P0: 일괄 처리 UX 개선

### 기능 테스트

- [ ] **ETA 표시**
  1. 매물 5개 이상 선택
  2. "일괄 승인" 클릭
  3. 진행 다이얼로그에서 "남은 시간" 표시 확인
  4. "X/초" 처리 속도 표시 확인
  5. ETA가 실시간으로 업데이트되는지 확인

- [ ] **세그먼트 진행바**
  1. 진행바가 성공(녹색)/실패(빨강) 구간으로 나뉘는지 확인
  2. 파란색 인디케이터가 현재 처리 위치를 표시하는지 확인
  3. 백분율(%)이 하단에 표시되는지 확인

- [ ] **상태 카드**
  1. 성공/실패/대기 카드가 그라데이션 배경으로 표시되는지 확인
  2. 숫자가 `tabular-nums` 폰트로 정렬되어 보이는지 확인
  3. 처리 중일 때 성공/실패 카드에 펄스 애니메이션이 있는지 확인

- [ ] **실패 항목 재시도**
  1. 일괄 처리 중 일부 실패 발생 시 (네트워크 끊기 등으로 테스트)
  2. "실패한 항목 (N개)" 빨간 박스 표시 확인
  3. "재시도" 버튼 클릭
  4. 실패한 항목만 재처리되는지 확인

- [ ] **완료 메시지**
  1. 모든 항목 처리 완료 시 녹색 완료 메시지 표시 확인
  2. CheckCircle 아이콘 표시 확인
  3. 성공/실패 개수 요약 확인

## P1-A: Table Sorting

### 기능 테스트

- [ ] **정렬 헤더**
  1. "주간 가격", "AI 점수", "등록일" 헤더에 ChevronUp/Down 아이콘 확인
  2. 헤더 클릭 시 정렬 방향 변경 확인
  3. 활성 방향이 Primary 색상으로 강조되는지 확인

- [ ] **주간 가격 정렬**
  1. "주간 가격" 헤더 클릭 (오름차순)
  2. 낮은 가격 → 높은 가격 순 정렬 확인
  3. 다시 클릭 (내림차순)
  4. 높은 가격 → 낮은 가격 순 정렬 확인
  5. 숫자가 `tabular-nums`로 정렬되어 보이는지 확인

- [ ] **AI 점수 정렬**
  1. "AI 점수" 헤더 클릭 (기본: 내림차순)
  2. 높은 점수 → 낮은 점수 순 정렬 확인
  3. NULL 값이 하단에 위치하는지 확인

- [ ] **등록일 정렬**
  1. "등록일" 헤더 클릭 (기본: 최신순)
  2. 최신 → 오래된 순 정렬 확인
  3. 날짜가 `mono` 폰트로 일관되게 보이는지 확인

### 예상 결과

```

   ↓
300,000원
450,000원
600,000원

## P1-B: AI 스코어 시각화

### 기능 테스트

- [ ] **Inline Gauge**
  1. AI 점수 컬럼에서 게이지 바 확인
  2. 게이지 바 길이가 점수에 비례하는지 확인 (72점 = 72% 너비)
  3. 색상 확인:
     - 70점 이상: 에메랄드 (녹색)
     - 40-69점: 앰버 (주황)
     - 40점 미만: 레드 (빨강)

- [ ] **점수 뱃지**
  1. 숫자가 `tabular-nums` 폰트로 정렬되어 보이는지 확인
  2. `ring-1` 테두리가 점수 색상과 일치하는지 확인
  3. NULL 값일 때 "-" 표시 확인

- [ ] **애니메이션**
  1. 페이지 로드 시 게이지 바가 부드럽게 증가하는지 확인 (500ms)
  2. 정렬 변경 시 게이지 바가 부드럽게 변화하는지 확인

### 예상 결과

```
AI 점수

[━━━━] 25                   (빨강)
- -                         (NULL)

## P1-C: Row Animations

### 기능 테스트

- [ ] **승인 애니메이션**
  1. 매물 1개의 "승인" 버튼 클릭
  2. **Approving**: 행이 녹색 배경으로 변하며 미묘하게 펄스하는지 확인
  3. **Approved**: 녹색 glow가 나타나며 1.5초간 fade-out되는지 확인
  4. 행이 오른쪽으로 20px 이동하며 사라지는지 확인
  5. 1.5초 후 데이터 리로드되며 행이 제거되는지 확인

- [ ] **반려 애니메이션**
  1. 매물 1개의 "반려" 버튼 클릭
  2. 반려 사유 입력 후 "반려하기" 클릭
  3. **Rejecting**: 행이 빨간 배경으로 변하며 펄스하는지 확인
  4. **Rejected**: 빨간 glow가 나타나며 1.5초간 fade-out되는지 확인
  5. 행이 오른쪽으로 이동하며 사라지는지 확인

- [ ] **오류 애니메이션**
  1. 네트워크를 끊은 상태에서 "승인" 클릭 (개발자 도구 Offline 모드)
  2. **Error**: 행이 좌우로 흔들리는지 확인 (shake 애니메이션)
  3. 빨간 배경이 나타나는지 확인
  4. 2초 후 애니메이션이 제거되고 원래 상태로 돌아오는지 확인

- [ ] **Hover 상태**
  1. 애니메이션 중이 아닌 행에 마우스 hover
  2. `hover:bg-muted/50` 배경색 변경 확인
  3. 애니메이션 중인 행에는 hover 효과가 없는지 확인

### 예상 결과

```
Timeline:
승인 클릭 → 녹색 펄스 (1초) → 녹색 glow + fade-out (1.5초) → 데이터 리로드

반려 클릭 → 빨강 펄스 (1초) → 빨강 glow + fade-out (1.5초) → 데이터 리로드

오류 발생 → 좌우 흔들림 (0.6초) → 빨간 배경 (2초) → 원래 상태
```

## 접근성 (Accessibility) 테스트

### 키보드 네비게이션

- [ ] **Tab 키**
  1. Tab 키로 "주간 가격", "AI 점수", "등록일" 헤더 포커스 이동 확인
  2. Enter/Space로 정렬 토글 확인
  3. 포커스 링(`focus:ring-2`)이 명확히 보이는지 확인

### 스크린 리더 (선택 사항)

- [ ] **ARIA 속성**
  1. 정렬 헤더에 `aria-sort` 속성 확인 (개발자 도구)
  2. 진행바에 `role="progressbar"` 속성 확인

### Reduced Motion

- [ ] **prefers-reduced-motion**
  1. macOS: 시스템 환경설정 > 손쉬운 사용 > 디스플레이 > "움직임 줄이기" 활성화
  2. Windows: 설정 > 접근성 > 시각 효과 > 애니메이션 효과 끄기
  3. 페이지 새로고침 후 모든 애니메이션이 비활성화되는지 확인
